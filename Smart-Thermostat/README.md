# Smart Thermostat Final Project: CS:350

## Project Summary and Problem Being Solved

This project involved designing and implementing a smart thermostat prototype using a Raspberry Pi. The goal was to simulate how a real-world thermostat monitors temperature, responds to user input, and controls system behavior using embedded software. The project solves the problem of managing heating and cooling logic in a structured and reliable way while interacting directly with physical hardware components.

## What I Did Particularly Well

One area I performed particularly well was using a state machine to organize system behavior. Breaking the thermostat into off, heat, and cool states helped keep the logic clear and predictable. I also did well integrating multiple hardware components—such as the temperature sensor, buttons, LEDs, and LCD—into a single cohesive system while debugging issues incrementally instead of changing multiple things at once.

## Where I Could Improve

If I were to improve this project, I would spend more time planning and documenting the system earlier in the development process. While the final implementation works well, adding more detailed design notes and testing strategies earlier could have reduced debugging time and made future updates easier.

## Maintainability, Readability, and Adaptability

To make the project maintainable and readable, I used clear variable names, modular functions, and consistent formatting. State machine logic was separated from hardware setup where possible, making the system easier to understand and modify. This structure allows the thermostat to be adapted in the future to support additional sensors, outputs, or features without major redesign.
